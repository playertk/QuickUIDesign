## Custom Events

### `useQuickUIEventSender` and `useQuickUIEventListener`

Implement custom event communication between Web and UE5, allowing users to design their own event names and JSON formats.
- `useQuickUIEventSender` is used to send events, requires specifying event name (corresponds to backend QuickUIListenEvent reception). Sends via `emitjsonevent` with automatic fallback to `executejs`.
- `useQuickUIEventListener` is used to listen for events, requires specifying event name (corresponds to backend QuickUIEventCallBack sending)

```tsx
import { useQuickUIEventSender, useQuickUIEventListener } from 'ue-connect'

function EventSystemComponent() {
  // Send event - need to specify event name (corresponds to backend QuickUIListenEvent)
  const sendEvent = useQuickUIEventSender('playerUpdate')

  // Listen for event - listen for events from UE (corresponds to backend QuickUIEventCallBack)
  useQuickUIEventListener('gameStateChange', (event) => {
    console.log('Game state changed:', event)
    console.log('Event topic:', event.topic)
    console.log('Event payload:', event.payload)
    console.log('Timestamp:', event.timestamp)
    console.log('Event source:', event.source) // "web" or "ue"
  })

  const updatePlayer = () => {
    // Send event to UE (fire-and-forget)
    sendEvent({
      health: 100,
      ammo: 30,
      position: { x: 10, y: 20, z: 30 }
    })
  }

  return <button onClick={updatePlayer}>Update player data</button>
}
```

**useQuickUIEventSender parameter description:**

- `fnName`: Event name (corresponds to Blueprint / UMG event name)
- Return value: Event sending function, accepts `payload?: QuickEventPayload` parameter
- Sending mechanism: First attempts `ue.uecommand.emitjsonevent(fnName, raw)`, falls back to `ue.uecommand.executejs(js)` if unavailable

**useQuickUIEventListener Parameters:**

- `fnName`: Event name (corresponds to Blueprint / UMG event name)
- `handler`: Event handler function, receives `QuickEvent` object
  - `event.type`: Event type (fixed as "event"), auto-generated by the API, no user intervention required
  - `event.topic`: Event topic/name, auto-populated by the API based on the registered `fnName`, no user intervention required
  - **`event.payload`: Event payload data — the only field users need to focus on and design**, used to transmit business-level custom data (e.g., player info, game state, UI interaction data, etc.). Supports arbitrary JSON structures; users can freely define its shape based on business needs.
  - `event.timestamp`: Event timestamp, auto-generated by the API, no user intervention required
  - `event.source`: Event source ("web" or "ue"), auto-detected and populated by the API, no user intervention required

> **Core Design Philosophy**: `payload` is the true data entry point that users need to design and handle. The remaining fields (`type`, `topic`, `timestamp`, `source`) are all auto-generated by the API following conventions — users only need to understand their meaning without manual editing or intervention.


### QuickEvent Type

```tsx
interface QuickEvent<Payload = QuickEventPayload> {
  type: 'event' // Event type (fixed as "event")
  topic: string // Event topic/name
  payload?: Payload // Event payload data
  timestamp: number // Event timestamp
  source: 'web' | 'ue' // Event source
}

type QuickEventPayload = Record<string, any> | null
```

### QuickEvent Type

```tsx
import { useQuickUIEventSender, useQuickUIEventListener, type QuickEvent } from 'ue-connect'

// Define custom payload type
interface PlayerEvent {
  playerId: string
  health: number
  position: { x: number; y: number; z: number }
}

function TypedComponent() {
  // Use typed event
  const sendPlayerUpdate = useQuickUIEventSender('playerUpdate')

  // Listen for typed event
  useQuickUIEventListener('playerUpdate', (event: QuickEvent<PlayerEvent>) => {
    if (event.payload) {
      console.log('Player ID:', event.payload.playerId)
      console.log('Player health:', event.payload.health)
      console.log('Player position:', event.payload.position)
    }
  })

  const updatePlayer = () => {
    sendPlayerUpdate({
      playerId: 'player123',
      health: 100,
      position: { x: 10, y: 20, z: 30 }
    })
  }

  return <button onClick={updatePlayer}>Update player</button>
}
```
